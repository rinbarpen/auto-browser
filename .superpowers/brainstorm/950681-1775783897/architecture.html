Auto Browser 架构方向

基于现有 agent-browser 内核，新增本地控制服务，桌面客户端和插件共享同一任务执行协议

推荐基线: Shared Core + Local Control Service
Surface A

桌面客户端

任务创建、步骤预览、一次审批、执行中直播、历史记录、浏览器与 profile 选择

Surface B

浏览器插件

当前页面内控制台、上下文感知、人工接管、状态展示，不自带完整调度内核

Shared Core

本地控制服务

统一暴露任务 API、计划预览 API、执行事件流、权限与风险闸门、浏览器实例管理

Task Planner Approval Gate Execution Orchestrator Event Stream Browser Registry
Execution Layer

agent-browser Runtime

复用 session、profile、extension、streaming、CDP、浏览器拉起与连接能力

Targets

本地浏览器环境

Chrome / Edge / Chromium，可指定可执行文件和用户 profile 目录，插件首发 Chromium 系

核心交互

1

先审后执行

AI 先生成可读步骤列表，用户一次确认后再开始

2

执行中直播

执行时持续推送当前步骤、状态和浏览器画面，可人工接管

3

插件页内协作

当用户就在目标页面时，插件直接展示任务状态并提供接管入口
